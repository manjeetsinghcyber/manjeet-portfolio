Web Server Hardening (Apache & WordPress)

Cybersecurity Project by Manjeet Singh

Project Summary

In this project, I installed an Apache web server and hardened it by improving security settings. I also secured a WordPress installation by removing default files, updating permissions, and disabling dangerous features. This project shows real-world system administration and security skills.

Tools Used

  • Ubuntu/Linux
  • Apache2 Web Server
  • WordPress
  • Terminal & File Permissions

Steps Performed

1. Installed Apache

sudo apt update
sudo apt install apache2

2. Disabled Directory Listing

Edited Apache configuration to prevent attackers from listing files:

Options -Indexes

3. Set Correct File Permissions

Secured WordPress directory:

sudo chown -R www-data:www-data /var/www/html
sudo find /var/www/html -type d -exec chmod 755 {} \;
sudo find /var/www/html -type f -exec chmod 644 {} \;

4. Removed Default Apache Files

sudo rm /var/www/html/index.html

5. Blocked .htaccess and config file access

<FilesMatch "^(wp-config\.php|\.htaccess)$">
Require all denied
</FilesMatch>

6. Updated WordPress & Plugins

wp core update
wp plugin update --all
