MITRE ATT&CK Technique Simulation

Cybersecurity Project by Manjeet Singh

Project Summary

In this project, I simulated real attacker techniques by following the MITRE ATT&CK framework. I used simple and safe commands to understand how attackers gather information, move in a system, and execute actions. Then I mapped each step to a MITRE technique ID.

Framework Used

  • MITRE ATT&CK (Enterprise Matrix)
  • Linux Terminal
  • Windows Command Prompt / PowerShell

Techniques Simulated

1. Reconnaissance – T1082 (System Information Discovery)

Checked system info:

uname -a
systeminfo

2. Credential Access – T1110 (Brute Force – Safe Example)

Simulated a safe brute-force pattern using failed login attempts:

ssh wronguser@localhost

3. Discovery – T1083 (File and Directory Discovery)

Listed system directories:

ls -la /etc

4. Execution – T1059 (Command-Line Interface)

Used standard command execution:

bash -c "echo Simulation Test"

5. Persistence – T1037 (Login Script Modification)

Viewed login scripts (no changes made):

cat ~/.bashrc

6. Defense Evasion – T1070 (Clear Command History – SAFE, NO EXECUTION)

I studied this technique but did not execute history clearing.

What I Learned

I learned how attackers think and how cybersecurity teams use MITRE ATT&CK to track hacker behavior. I also learned how each command maps to a specific attack technique, which is extremely useful for SOC analysts and blue team work.
