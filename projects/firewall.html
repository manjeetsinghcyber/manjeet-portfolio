Firewall Rules & Traffic Filtering

Cybersecurity Project by Manjeet Singh

Project Summary

In this project, I configured basic firewall rules using Linux iptables and reviewed how pfSense handles rules. I learned how traffic is allowed or blocked, how ports work, and how security teams protect a network by creating clear rules.

Tools Used

  • Kali Linux / Ubuntu
  • iptables
  • pfSense (for rule examples)

Linux iptables Rules

1. List all firewall rules:

sudo iptables -L -v -n

2. Allow SSH (port 22):

sudo iptables -A INPUT -p tcp --dport 22 -j ACCEPT

3. Allow HTTP (port 80):

sudo iptables -A INPUT -p tcp --dport 80 -j ACCEPT

4. Allow HTTPS (port 443):

sudo iptables -A INPUT -p tcp --dport 443 -j ACCEPT

5. Allow ping (ICMP):

sudo iptables -A INPUT -p icmp -j ACCEPT

6. Block everything else:

sudo iptables -P INPUT DROP

7. Save rules (Debian/Ubuntu):

sudo netfilter-persistent save

pfSense Firewall Rules (Explained)

  • LAN → WAN allowed (default)
  • Blocked all inbound WAN traffic
  • Allowed DNS (53), HTTP(80), HTTPS(443)
  • Allowed ICMP Echo (for ping)
  • Enabled logging on all blocked traffic

pfSense rules work from top to bottom. The first match wins, so ordering rules correctly is very important.

What I Learned

I learned how firewalls control network traffic using rules and policies. I also learned the difference between inbound and outbound rules, why block-all-allow-some is safer, and how to open specific ports for services. This project helped me understand real network security.
