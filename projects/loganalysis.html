Linux & Windows Log Analysis

Cybersecurity Project by Manjeet Singh

Project Summary

In this project, I analyzed Linux and Windows logs to identify suspicious activity. I looked for failed login attempts, brute-force patterns, system errors, and unusual behavior. Log analysis is an important part of SOC and Blue Team work.

Tools Used

  • Linux (auth.log, syslog)
  • Windows Event Viewer (Security Logs)
  • grep / awk / cat
  • Event ID analysis

Linux Log Analysis

Checked authentication logs:

cat /var/log/auth.log

Searched for failed logins:

grep "Failed password" /var/log/auth.log

Searched for successful SSH logins:

grep "Accepted password" /var/log/auth.log

Viewed general system logs:

cat /var/log/syslog

Windows Log Analysis

Opened Event Viewer:

Windows Logs → Security

Important Event IDs:

  • 4624 – Successful Login
  • 4625 – Failed Login
  • 4634 – Logoff
  • 4688 – New Process Created
  • 4720 – User Account Created

Example suspicious pattern:

Multiple 4625 failed logins → possible brute force

What I Learned

I learned how to read system logs, identify brute-force attempts, track user logins, and understand Windows Event IDs. This project helped me understand real SOC analysis work and how security teams investigate incidents.
